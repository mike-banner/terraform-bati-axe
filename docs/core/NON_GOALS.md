# NON GOALS

> Ce fichier est aussi important que le MASTER_PLAN.
> Savoir ce qu'on ne fait PAS évite la dispersion et la dette.

---

## Ce que BÂTI-AXE n'est PAS

| Ce n'est PAS | Ce que c'est |
| :--- | :--- |
| Un réseau social artisans | Une plateforme de matching sélectif |
| Un ERP chantier | Un système de génération de leads qualifiés |
| Un marketplace freelancing ouvert | Un filtre de qualité avec validation |
| Une app temps réel complexe | Une plateforme asynchrone et fiable |
| Une plateforme low-cost | Une offre premium basée sur la confiance |
| Un annuaire classique | Un système de vérification actif |

---

## Ce que le projet refuse (pour toujours)

| Refus | Justification |
| :--- | :--- |
| Croissance anarchique (tout le monde peut s'inscrire) | Détruit la proposition de valeur confiance |
| Validation faible des artisans | Risque légal + destroy brand |
| Dette technique rapide pour "aller vite" | Coût x10 à refactorer plus tard |
| Stack expérimentale non éprouvée | Risque opérationnel inacceptable |
| Architecture suringénierée (microservices Phase 1) | Ralentit, complexifie, sans bénéfice réel |
| Paiements entre particuliers et artisans | Réglementation PSP complexe, hors scope |
| Messagerie in-app avancée Phase 1 | Email suffit, feature creep |

---

## Ce qui est hors scope DÉFINITIVEMENT

- Gestion de chantier (planning, suivi travaux)
- Gestion RH pour les artisans
- Comptabilité chantier
- Appels d'offres publics
- Formation professionnelle
- Garanties financières chantier

---

## Ce qui est hors scope POUR L'INSTANT

- Application mobile native → Phase 2 (PWA acceptable Phase 1)
- Chat temps réel → Phase 2
- Automatisation validation par IA → Phase 2
- API publique pour partenaires → Phase 2
