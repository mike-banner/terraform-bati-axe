**Admin Console (admin/index.vue):**

1. **Table Container:**
   - Background: white
   - Border: 1px solid #E2E8F0
   - Border-radius: 12px (--radius-lg)
   - Overflow: auto (horizontal scroll on mobile if needed, but prefer responsive)
   - Box-shadow: shadow-sm

2. **Table Header (thead):**
   - Background: #ECFEFF (light cyan)
   - Font: Figtree 600, 12px uppercase, dark cyan (#164E63)
   - Padding: 12px vertical, 16px horizontal (--space-md)
   - Text-align: left (or center for numeric columns)
   - Border-bottom: 1px solid #E2E8F0

3. **Table Rows (tbody tr):**
   - Height: 48px (uniform row height for alignment)
   - Border-bottom: 1px solid #E2E8F0
   - Transition: all 200ms ease

4. **Table Cells (td):**
   - Font: Noto Sans 400, 14px, dark gray (#374151)
   - Padding: 12px vertical, 16px horizontal (--space-md)
   - Vertical-align: middle (for proper cell centering)
   - Overflow: auto (for long text)

5. **Table Row Hover:**
   - Background: #F0FEFF (very light cyan, almost white but slightly tinted)
   - Box-shadow: shadow-sm (lift effect)
   - Transition: 200ms ease

6. **Action Buttons (per row or grouped):**
   - Container: `display: flex; gap: 8px; --space-sm`
   - Edit button: secondary style (cyan outline, border #0891B2, text #0891B2, 10px 16px padding)
   - Delete button: danger style (red border #DC2626, red text #DC2626, 10px 16px padding)
   - Approve button: primary style (green bg #22C55E, white text, 10px 16px padding)
   - Focus: outline ring visible
   - Hover: opacity-90, no scale

7. **Modals (if used for edit/delete confirmation):**
   - Overlay: rgba(0,0,0,0.5), backdrop-filter blur(4px)
   - Modal: white bg, 16px radius, shadow-xl, 32px padding
   - Modal heading (h2): Figtree 600, 20px
   - Modal body: Noto Sans 400, 14px, line-height 1.6
   - Modal buttons: group at bottom, 8px gap

8. **Responsive Admin:**
   - Desktop (1024px+): full table layout as described
   - Tablet (768px): horizontal scroll table (acceptable for internal tool)
   - Mobile (375px): stacked card layout per row (optional, or horizontal scroll)

**Legal Pages (all three: mentions-legales, cgu, confidentialite):**

1. **Page Container:**
   - Max-width: 800px (--max-w-3xl)
   - Margin: auto
   - Padding: 48px horizontal (--space-2xl) mobile, 64px desktop (--space-3xl)

2. **Page Heading (h1):**
   - Font: Figtree 700, 32px-48px clamp
   - Color: #164E63 (dark cyan)
   - Margin-bottom: 48px (--space-2xl)
   - Text-wrap: balance

3. **Section Headings (h2, h3):**
   - h2: Figtree 600, 24px, dark cyan, margin-top 48px (--space-2xl), margin-bottom 16px (--space-md)
   - h3: Figtree 500, 18px, dark cyan, margin-top 32px (--space-xl), margin-bottom 12px (--space-sm)

4. **Body Text (p):**
   - Font: Noto Sans 400, 16px, line-height 1.6, dark gray (#374151)
   - Margin-bottom: 16px (--space-md)
   - Max-width: 70ch (reading line length for comfort)

5. **Lists (ul, ol):**
   - Font: Noto Sans 400, 14px-16px, dark gray
   - List-style: disc (ul) or decimal (ol)
   - Margin-left: 24px (indent)
   - Margin-bottom: 16px (--space-md)
   - List-item: margin-bottom 8px (--space-sm)

6. **Links (a):**
   - Font: inherit from context
   - Color: #0891B2 (primary cyan)
   - Text-decoration: none (default), underline on hover
   - Focus: outline 2px solid #0891B2, outline-offset 2px

7. **Emphasis (strong, em):**
   - strong: Noto Sans 600 (bold)
   - em: italic, no color change

8. **Code (if used for data/examples):**
   - Font: monospace (e.g., Courier New, Fira Code)
   - Background: #F3F4F6 (light gray)
   - Padding: 2px 4px
   - Border-radius: 4px

9. **Blockquotes (if used for highlights):**
   - Border-left: 4px solid #0891B2 (cyan)
   - Padding-left: 16px (--space-md)
   - Background: #ECFEFF (light cyan, subtle)
   - Margin: 24px 0 (--space-lg)

10. **Responsive Legal:**
    - Mobile (375px): single column, smaller padding 24px (--space-lg), text-align justified (optional)
    - Tablet (768px+): single column, full padding 64px (--space-3xl), max-width container

11. **Typography Audit:**
    - All headings: Figtree (600, 700)
    - All body: Noto Sans (400, 500 for emphasis)
    - Semantic HTML: h1 > h2 > h3, proper list structure, links with descriptive text

12. **Avoid Anti-Patterns:**
    - ❌ No emoji bullets (use standard •)
    - ❌ No missing focus states on links
    - ❌ No low contrast links (cyan on white passes 4.5:1)
    - ❌ No hardcoded colors in legal text (use CSS classes)
    - ❌ No unstyled headings (proper hierarchy)

13. **Accessibility:**
    - Legal pages: proper heading hierarchy (h1 once per page)
    - Links: descriptive text (not "click here", e.g., "Read our privacy policy")
    - Lists: semantic ul/ol tags
    - Focus: all interactive elements have visible focus ring
