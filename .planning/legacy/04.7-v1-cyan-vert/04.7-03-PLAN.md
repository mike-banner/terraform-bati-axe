  <name>Task 1: Refactor form inputs, labels, selects, textareas to MASTER.md input spec</name>
  <files>app/pages/pro/claim.vue, app/pages/pro/profile.vue, app/pages/espace/profil.vue</files>
  <action>
Refactor form styling across all three pro form pages. Apply unified input/label/select styles:

**Input Element Styles (all input[type], select, textarea):**
- Base: `padding: 12px 16px; border: 1px solid #E2E8F0; border-radius: 8px; font-size: 16px; font-family: 'Noto Sans', sans-serif; transition: border-color 200ms ease, box-shadow 200ms ease; background: white;`
- Focus: `border-color: #0891B2; outline: none; box-shadow: 0 0 0 3px #0891B220;`
- Error state: Add class to input wrapper: `border-red-500 border-2` and text `text-red-600`
- Disabled: `background: #F1F5F9; color: #94A3B8; cursor: not-allowed;`
- Hover (non-focus): `border-color: #CBD5E1` (slight darkening)

**Label Styling:**
- Font: Figtree, font-weight 500, 14px
- Color: #164E63 (dark cyan, matches --color-text)
- Margin-bottom: 8px (--space-sm)
- Use `<label for="input-id">` for accessibility

**Section Separators:**
- Between form sections: `border-top: 1px solid #E2E8F0; margin-top: 48px; padding-top: 48px;`

**File Upload Inputs (Kbis, Décennale):**
- Border-style: 2px dashed #0891B2 (indicate dropzone)
- Padding: 24px (--space-lg)
- Text: Noto Sans 14px
- Hover: bg-light-cyan (#ECFEFF)
- Focus: Border solid cyan, ring

**Checkboxes (SMS opt-in, CGU accept):**
- Size: 20px x 20px
- Border: 1px solid #E2E8F0
- Checked: bg-cta (#22C55E), border-cta
- Focus: ring-primary/20
- Label: adjacent text, Noto Sans 14px

**Radio Buttons (if category selection uses radios):**
- Same as checkboxes; for multi-select categories, prefer checkboxes or pill buttons

**Select Dropdowns:**
- Same padding/border/focus as input
- Arrow icon: default browser style, or use SVG if custom styling needed
- Focus: same cyan ring

**Responsive Layout:**
- Mobile: single column, 100% width forms
- Tablet (768px+): two-column layout for non-blocking fields (e.g., first_name + last_name side-by-side)
- Desktop (1024px+): three-column possible for short fields

**Form Section Grouping (pro/claim example):**
1. Auth section: email, password fields
2. Company section: company_name, SIRET, postal_code
3. Profile section: full_name, phone, bio
4. Categories section: checkboxes for multi-select
5. Documents section: file uploads (Kbis, Décennale)
6. Consent section: SMS opt-in, CGU checkbox
7. Submit button: green, full width on mobile, auto width on desktop

**Per-File Refactors:**

*/pro/claim.vue:*
- Form container: `<form @submit.prevent="submitClaim" class="max-w-2xl mx-auto px-6 py-12">`
- All input[type="text"], input[type="email"], input[type="password"], select: apply unified input class
- Labels: apply Figtree label class
- Error messages (globalError, fieldErrors): display in red (#DC2626)
- File input: use dashed border, drag-and-drop zone styling
- Submit button: "Je revendique mon profil" → green primary button, full width mobile, auto desktop

*/pro/profile.vue:*
- Same input/label/select styling as claim
- Profile display section (if exists): separate from form, use card styling (light cyan bg, shadow)
- Avatar/logo upload: dashed border, similar to Kbis upload
- Bio textarea: 4-5 rows, 16px font, same focus/border states
- Submit button: "Enregistrer" or "Mettre à jour" → secondary or green (per UX)

*/espace/profil.vue:*
- Likely dashboard profile management
- Same form styles
- May have sections for different profile aspects (bio, skills, contact)
- Each section: separator line, subsection heading (Figtree 500, 16px)

**Typography Audit:**
- Form labels: Figtree 500, 14px
- Input text: Noto Sans 400, 16px
- Help text: Noto Sans 400, 12px, #6B7280
- Error messages: Noto Sans 500, 14px, #DC2626
- Section headings: Figtree 600, 18px

**Avoid Anti-Patterns:**
- ❌ No placeholder-as-label (use actual labels)
- ❌ No instant form submissions (use loading spinner during submit)
- ❌ No hidden required field indicators (use asterisk or "required" text)
- ❌ No unstyled error messages (must be visible, red, with icon if possible)
- ❌ No missing focus states on form inputs (all must have visible ring or outline)

**Accessibility:**
- All inputs must have associated labels (label for="id" or wrap input in label)
- Error messages linked to inputs via aria-describedby
- Required fields marked with aria-required="true"
- Fieldsets group related inputs (if using radio/checkbox groups)
