  <name>Task 1: Rewrite tailwind.css — Fonts, colors, tokens</name>
  <files>app/assets/css/tailwind.css</files>
  <action>
Replace the current tailwind.css (Clash Display + Geist Variable, OKLCH Brique & Béton) with a new foundation:

1. Replace @font-face declarations:
   - Remove Clash Display + Geist Variable references
   - Add @import url Google Fonts (Figtree:300,400,500,600,700 + Noto Sans:300,400,500,700)
   - Keep font-display: swap for performance

2. Update @theme tokens:
   - --font-sans: 'Noto Sans', sans-serif
   - --font-heading: 'Figtree', sans-serif
   - Keep other shadcn-vue tokens (sidebar, chart, etc.) unchanged for now

3. Rewrite :root color variables to MASTER.md palette (hex direct, not OKLCH):
   - --color-primary: #0891B2 (cyan, primary links/secondary CTAs)
   - --color-secondary: #22D3EE (cyan lighter, hover states)
   - --color-cta: #22C55E (green, primary buttons)
   - --color-background: #ECFEFF (light cyan page bg)
   - --color-text: #164E63 (dark cyan text)
   - --color-border: #E2E8F0 (gray border)
   - --color-border-error: #DC2626 (red error state)
   - --color-border-disabled: #F1F5F9 (light gray disabled state)
   - --color-ring: #0891B220 (cyan with 13% opacity for focus ring)

4. Remove the Brique & Béton variables (brand, brand-hover, brand-foreground, brand-ink, surface, surface-sable, border-strong, etc.) — they are no longer used in Phase 4.7+.

5. Add spacing tokens to @theme (explicit token names for grid consistency):
   - --space-xs: 4px / 0.25rem
   - --space-sm: 8px / 0.5rem
   - --space-md: 16px / 1rem
   - --space-lg: 24px / 1.5rem
   - --space-xl: 32px / 2rem
   - --space-2xl: 48px / 3rem
   - --space-3xl: 64px / 4rem

6. Add shadow tokens:
   - --shadow-sm: 0 1px 2px rgba(0,0,0,0.05)
   - --shadow-md: 0 4px 6px rgba(0,0,0,0.1)
   - --shadow-lg: 0 10px 15px rgba(0,0,0,0.1)
   - --shadow-xl: 0 20px 25px rgba(0,0,0,0.15)

7. Add radius tokens:
   - --radius-sm: 6px
   - --radius-md: 8px
   - --radius-lg: 12px
   - --radius-xl: 16px

8. Keep @layer base and body defaults (bg-background, text-foreground, font-sans).

9. Add component base styles in @layer components (or keep as utility references for now):
   - .btn-primary: bg-cta text-white, 12px 24px padding, radius-md, hover opacity-90 + translateY-1, focus ring, 200ms transition
   - .btn-secondary: bg-transparent border-2 border-primary text-primary, same padding/radius/hover/focus
   - .card: bg-background border border-border radius-lg p-6, shadow-md, hover shadow-lg + translateY-2
   - .input: border border-border radius-md, focus border-primary ring-primary/20, disabled bg-border-disabled
   - .modal: bg-white radius-xl p-8 shadow-xl max-w-md

10. Note: Tailwind's arbitrary values (e.g., `bg-[#0891B2]`) are still valid fallback if a color is not in the theme, but all MASTER.md colors MUST be in :root so pages can use semantic class names (bg-primary, text-text, etc.).
