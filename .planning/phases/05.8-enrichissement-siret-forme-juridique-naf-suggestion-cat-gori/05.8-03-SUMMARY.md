# Phase 05.8 Plan 03: Auto-approbation Kbis conditionnee au SIRET actif Summary

Auto-approuver le Kbis a l'upload quand `professionals.siret_status === 'active'`, evitant une revue admin redondante quand l'API gouv a deja confirme l'entreprise au claim.

## What Was Built

Dans `server/api/v1/pro/documents/upload.post.ts` : avant de calculer le statut d'insertion, lecture de `siret_status` du pro courant (`professionals.select('siret_status').eq('id', uid)`). Nouvelle variable `isKbisAutoApproved = document_type === 'kbis' && proRow?.siret_status === 'active'`. Le statut d'insertion devient `'approved'` si `isDecennale || isKbisAutoApproved`, sinon `'pending'`. Un Kbis auto-approuve recoit `reviewed_at` dans l'insert (coherent avec un document deja statue), sans les champs specifiques decennale (`policy_number`, `expiry_date`) ni modification des labels/`decennal_status`. Le retour JSON reflete l'auto-approbation via `approved: isDecennale || isKbisAutoApproved`.

Le bloc d'auto-approbation decennale (mise a jour `professionals.labels` / `decennal_status`) est inchange.
