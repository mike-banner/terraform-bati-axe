## Spacing Scale

Reuse the exact tokens already declared in `app/assets/css/tailwind.css` (`--space-*`). Do not introduce new spacing values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-xs`) | Icon-to-label gaps inside tiles |
| sm | 8px (`--space-sm`) | Compact spacing between chips/badges |
| md | 16px (`--space-md`) | Default gap between tiles in a step |
| lg | 24px (`--space-lg`) | Padding inside tile cards, gap between step sections |
| xl | 32px (`--space-xl`) | Gap between step header and tile grid |
| 2xl | 48px (`--space-2xl`) | Vertical rhythm between major page sections (progress bar → step content) |
| 3xl | 64px (`--space-3xl`) | Top/bottom page padding on desktop only |

Exceptions:
- Touch targets for tuiles (pièces/type de travaux) on mobile: minimum 44px height enforced regardless of 4px-grid, per accessibility floor — apply as `min-h-11` (44px) on all selectable tiles.
- Lead Wall final tile: no exception, same tokens apply, treated as a regular Bento tile (per D-02, no modal/overlay).

## Typography

Reuse the DM Sans scale already live in the project. No new sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (tile description, helper text) | 16px | 400 | 1.5 |
| Label (tile title, form field label) | 18px | 700 | 1.3 |
| Heading (step title, e.g. "Quel type de rénovation ?") | 28px | 900 | 1.2 |
| Display (estimation result, e.g. "15 000 € – 22 000 €") | 40px (mobile) / 56px (desktop, `clamp`) | 900 | 1.1 |

Only 4 sizes, 2 weights (400 regular, 700/900 bold family — treat 700 and 900 as the "bold" register consistent with 04.7's heading/subtitle split). Do not add a 5th size or a 3rd weight for this phase.

## Interaction & Layout Contract (phase-specific, extends 04.7 patterns)

- **Navigation model**: tuiled Bento steps, one step visible at a time (not a scrolling all-in-one page). Progress bar at top shows N/6 (6 steps per SPEC.md, culminating in Lead Wall as step 6 — not a 7th overlay).
- **Transitions between tiles/steps**: Vue `<Transition>` with the existing `.reveal-item` stagger keyframes (`reveal-up`, `0.6s cubic-bezier(0.16, 1, 0.3, 1)`) already defined in `app/assets/css/tailwind.css` — reuse, do not invent new easing curves. Respect `prefers-reduced-motion` per 04.7 accessibility invariant.
- **Tile selection state**: single-select tiles (type de rénovation, surface bracket, gamme) get radio-like behavior; multi-select tiles (pièces concernées, Step 2) get checkbox-like behavior — both rendered as `.bento-card` with `border-orange-500 bg-orange-50` when active, matching the existing `.bento-card:hover` elevation defined at `tailwind.css:317-325`.
- **Lead Wall (D-02)**: implemented as the final tile in the same Bento flow — same card shell, same progress bar position (fully filled), NOT a modal/dialog/overlay. Name/email/phone inputs reuse existing form input components from `app/components/ui/*` and the same `RE_EMAIL`/`RE_PHONE_FR` validation regex already in `simulateur.vue`.
- **Result reveal**: estimation range (`15 000 € - 22 000 €`) is computed client-side (D-01) and only rendered AFTER successful lead wall submission — never shown before contact info is captured.
- **Mobile-first**: single column tile grid on mobile (`grid-cols-1`), 2-3 columns on `md:`+ breakpoints for icon-tiles (pièces/type de travaux). Max 5 clicks/steps to result is the UX target per CONTEXT.md — do not add sub-steps beyond the 6 declared in SPEC.md.
