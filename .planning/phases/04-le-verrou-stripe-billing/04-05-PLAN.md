```typescript
import Stripe from 'stripe'

// MANDATORY: Cloudflare Pages uses Fetch API, not Node.js http
const stripe = new Stripe(config.stripeSecretKey as string, {
  httpClient: Stripe.createFetchHttpClient(),
})

// MANDATORY for webhook: SubtleCryptoProvider (WebCrypto async, not Node.js sync)
const webCrypto = Stripe.createSubtleCryptoProvider()

// MANDATORY: readRawBody NOT readBody (readBody parses JSON, invalidates Stripe signature)
const rawBody = await readRawBody(event)

// MANDATORY: constructEventAsync NOT constructEvent (sync version crashes on Cloudflare V8)

  rawBody,
  signature,
  config.stripeWebhookSecret as string,
  undefined,
  webCrypto  // 5th parameter — REQUIRED
)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe → POST /api/v1/stripe/webhook | Untrusted external caller — must verify Stripe signature before processing |
| user → POST /api/v1/stripe/checkout | Authenticated user — must verify identity before creating Stripe session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12 | Spoofing | POST /api/v1/stripe/webhook | mitigate | constructEventAsync + SubtleCryptoProvider verifies Stripe HMAC signature (timing-safe, replay protection via 5-min timestamp tolerance built into SDK) |
| T-04-13 | Tampering | webhook metadata.pro_id | mitigate | pro_id comes from metadata set by checkout.post.ts (server-side, uses user.id from auth session — not from client body) |
| T-04-14 | Information Disclosure | webhook handler | mitigate | No customer data logged; only stripeEvent.type and update result |
| T-04-15 | Elevation of Privilege | POST /api/v1/stripe/checkout | mitigate | serverSupabaseUser FIRST line → 401; session metadata.pro_id = user.id (not from request body) |
| T-04-16 | Denial of Service | webhook repeated events | accept | Stripe webhook retries are idempotent — UPDATE WHERE id/stripe_customer_id is safe to repeat |
| T-04-SC | Tampering | npm installs | accept | stripe package already installed and audited [OK] |
</threat_model>
