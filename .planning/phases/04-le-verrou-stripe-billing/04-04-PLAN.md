  <action>
  Create server/api/v1/leads/[id]/claim.patch.ts. This is a PATCH endpoint (no request body needed — action is implicit from the route).

  Flow:
  1. `const user = await serverSupabaseUser(event)` — FIRST line; throw 401 if missing
  2. `const id = getRouterParam(event, 'id')` → 400 if missing
  3. `const supabase = await serverSupabaseServiceRole(event) as any`
  4. Fetch pro profile: `.from('professionals').select('id, subscription_status').eq('id', user.id).single()` → 404 if not found
  5. Premium check: if `pro.subscription_status !== 'active'` → throw 403 with statusMessage: 'Réservé aux pros Premium.' — only active Premium subscribers can claim leads
  6. Fetch the lead to verify ownership: `.from('leads').select('id, status, pro_id').eq('id', id).eq('pro_id', user.id).single()` → 404 if not found (protects against cross-pro access AND non-existent leads)
  7. If `lead.status === 'claimed'` → throw 409 with statusMessage: 'Ce lead est déjà attribué.' — idempotence guard
  8. UPDATE: `.from('leads').update({ status: 'claimed' }).eq('id', id).eq('pro_id', user.id)` — double ownership check on update
  9. Return: `{ claimed: true, lead_id: id }`
  </action>
