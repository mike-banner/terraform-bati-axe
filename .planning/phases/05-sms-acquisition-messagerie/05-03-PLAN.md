# Phase 5 — Plan 03 : Feedback loop lead + Email onboarding

> Objectif : fermer la boucle côté particulier (relancer la recherche) et poser les emails d'accueil pro derrière un flag off par défaut. Couvre REQ-06 et REQ-07.

## Existant réutilisé
- `app/pages/mon-projet/[token].vue` — Espace Client (magic link).
- `server/api/v1/magic-link/[token].get.ts` — sert projet + pros + messages.
- `server/api/v1/leads/[id]/claim.patch.ts` — porte le **cap concurrentiel (3 max)**.
- `server/api/v1/market-local.get.ts` — marché en pull (projets visibles selon le cap).
- `server/utils/email.ts` — `sendEmail`.

## Partie A — Feedback loop (REQ-06)

### A1. Endpoint de relance
- **Fichier** : `server/api/v1/magic-link/[token]/relaunch.post.ts` (auth par `access_token` du projet, comme le reste de l'espace client).
- **Action** : ne réouvre que si le **cap de 3 est atteint**. Réouverture = remettre le projet visible dans `market-local` pour de nouveaux pros (mécanisme : selon l'implémentation du cap — flag projet `relaunched_at`/compteur, ou reset de la condition de visibilité). **Conserver** les `leads` et `messages` existants (historique intact).
- **Garde-fou** : idempotent / anti-spam (pas de relance multiple en rafale — cooldown ou borne).

### A2. UI bouton
- **Fichier** : `app/pages/mon-projet/[token].vue`.
- **Action** : afficher « Je n'ai pas trouvé mon bonheur → relancer ma recherche » **uniquement** quand le cap de 3 est atteint et aucune mise en relation aboutie. Appel POST relaunch + feedback visuel.

## Partie B — Email onboarding pro (REQ-07)

### B1. Flag d'activation
- **Fichier** : `.env.example` + `nuxt.config.ts` (`runtimeConfig`).
- **Action** : variable `ONBOARDING_EMAILS` (défaut `false`). Documenter dans `.env.example` (règle CLAUDE.md). Préfixe `NUXT_` côté Cloudflare pour le runtimeConfig.

### B2. Déclenchement
- **Fichier** : `server/api/v1/pro/claim.post.ts` (accueil au claim) et/ou `server/api/v1/admin/approve-pro.post.ts` (accueil à la validation).
- **Action** : si flag on → `sendEmail` de bienvenue (contenu texte simple, design plus tard). Si off → no-op. Email idempotent (ne pas renvoyer si déjà envoyé — marqueur type `onboarding_email_sent_at` sur `professionals`).

## Vérification (DoD)
- [ ] Relaunch réouvre le projet seulement si cap atteint ; leads/messages conservés.
- [ ] Relaunch idempotent (pas de spam de relance).
- [ ] Bouton visible uniquement dans la bonne condition.
- [ ] `ONBOARDING_EMAILS=false` → zéro email ; `true` → un seul email par pro.
- [ ] `.env.example` à jour.
- [ ] Test unitaire sur la condition « cap atteint » et sur le gate du flag onboarding.
