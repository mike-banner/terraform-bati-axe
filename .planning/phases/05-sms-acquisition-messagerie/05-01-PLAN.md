# Phase 5 - Plan 01: Magic Link & Messagerie In-App

## 1. Verrou Concurrentiel (Cap à 3)
- **Fichier** : `server/api/v1/leads/[id]/claim.patch.ts`
- **Action** : Compter les lignes dans `leads` pour ce `project_id`. Si >= 3, rejeter le déblocage (403).

## 2. Espace Client (Magic Link)
- **Fichier** : `server/api/v1/magic-link/[token].get.ts`
- **Action** : API publique qui retourne le projet, les pros ayant débloqué, et l'historique des messages.
- **Fichier** : `app/pages/mon-projet/[token].vue`
- **Action** : UI du particulier (statut, artisans, chat).

## 3. Messagerie API
- **Fichiers** : `server/api/v1/messages/index.post.ts` et `server/api/v1/messages/client.post.ts`.
- **Action** : Insertion dans `messages`. Utilisation d'un utilitaire `server/utils/email.ts`.

## 4. Interface Pro
- **Fichiers** : `app/pages/espace/leads/[id].vue` et `app/pages/espace/messages/index.vue`.
- **Action** : Intégration du composant de chat sur la fiche du chantier et page dédiée pour mobile.

## 5. Intégration Resend
- **Fichiers** : `server/utils/email.ts`, `.env`, `nuxt.config.ts`.
- **Action** : Ajout du SDK Resend. Mode hybride : `console.log` en environnement DEV, vrai call API en environnement PROD.
