## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (gap-1) | Icon gaps, inline badge padding |
| sm | 8px (gap-2, p-2) | Compact row gaps, badge inner, qualification badge strip |
| md | 16px (gap-4, px-4) | Card padding, form field gaps |
| lg | 24px (gap-6, px-6) | Page horizontal padding, section gaps, card inner rows |
| xl | 32px (mb-8) | Block separations within a page |
| 2xl | 40px (mb-10) | Page header bottom margin |
| 3xl | 64px (py-16) | Page vertical padding |

Exceptions:
- Card inner rows: `px-6 py-3` (24px / 12px) — inherits Phase 4 pattern
- Card inner rows with more content: `px-6 py-4` — inherits Phase 4 pattern
- Logo upload preview: 80×80px fixed square (not on scale — visual affordance for image)

## Typography

All text uses Geist Variable loaded via `font-sans`. No import required.

| Role | Classes | Size | Weight | Line Height |
|------|---------|------|--------|-------------|
| Page title | `text-3xl font-semibold tracking-tight text-foreground` | 30px | 600 | 1.2 |
| Hero display (Premium page) | `text-3xl font-semibold tracking-tight text-foreground` + `style="text-wrap: balance"` | 30px | 600 | 1.2 |
| Price display (Premium card) | `text-4xl font-semibold tracking-tight text-foreground` | 36px | 600 | 1.2 |
| Section header | `text-xs font-medium text-muted-foreground tracking-widest uppercase` | 12px | 500 | 1.5 |
| Card label / item label | `text-sm font-semibold text-foreground` | 14px | 600 | 1.5 |
| Body / descriptions | `text-sm text-muted-foreground` | 14px | 400 | 1.5 |
| Meta / timestamp / fine print | `text-xs text-muted-foreground` | 12px | 400 | 1.5 |

Declared sizes: 12px, 14px, 30px, 36px (4 sizes — within 3–4 max rule).
Declared weights: 400 (regular) + 600 (semibold — all headings, labels, CTAs).

> Note: `font-medium` (500) on section headers is a display-only utility class for the tracking/uppercase pattern inherited from Phase 4. The two operative weights are 400 and 600.

## Color

The site is **monochrome pure** with a single amber accent for Premium/warning states.
Token values from `app/assets/css/tailwind.css` OKLCH variables.

| Role | Token / Value | Usage |
|------|--------------|-------|
| Dominant surface (60%) | `bg-background` — oklch(1 0 0) | Page body, card backgrounds |
| Secondary surface (30%) | `bg-muted` — oklch(0.97 0 0) | Muted sections, skeleton, hover states |
| Accent Premium (10%) | `border-amber-300 bg-amber-50 text-amber-700` | See reserved list below |
| Destructive | `text-destructive` — oklch(0.577 0.245 27.325) | Form validation errors only |
| Border | `border-border` — oklch(0.922 0 0) | All cards, dividers, inputs |
| Text primary | `text-foreground` — oklch(0.145 0 0) | Headings, labels, values |
| Text secondary | `text-muted-foreground` — oklch(0.556 0 0) | Descriptions, meta, icons |

**Amber accent reserved exclusively for:**
1. Premium badge and Premium banner on leads page (paywall trigger)
2. Pricing card border on `/espace/premium`
3. Flouté badge on lead cards (locked state)
4. Locked contact section box on lead detail page
5. Free trial callout on `/espace/premium` (new in 4.5)
6. Paywall card shown at 4th lead (new in 4.5)

**What NOT to use:**
- No `text-sky-*`, `text-emerald-*`, `text-slate-*`
- No `bg-emerald-50`, `bg-slate-100` on cards
- No CSS `filter: blur()` — masking is server-side, display `*** *** ***` in `font-mono`
- No color for qualification badge icons beyond `text-foreground` (pass) / `text-muted-foreground` (fail)

## Global Consistency Rules

Inherit all rules from Phase 4 UI-SPEC. Extensions for Phase 4.5:

1. **Container:** `max-w-2xl mx-auto px-6 py-16` — identical across all espace pages
2. **Cards:** `border border-border rounded-lg divide-y divide-border` — no exceptions
3. **Section headers:** `text-xs font-medium text-muted-foreground tracking-widest uppercase mb-4`
4. **Inputs / textareas / selects:** `border border-border rounded-md text-sm bg-background focus:outline-none focus:ring-1 focus:ring-foreground/20`
5. **Read-only fields:** `bg-muted cursor-not-allowed text-muted-foreground` — visually distinct from editable
6. **Qualification badges:** icons only, no text labels on the card, `title` attribute for hover tooltip
7. **Amber constraint:** do NOT use amber on the `/espace/profil` page — no locked state there
8. **Counters:** character count on bio textarea always right-aligned, `text-xs text-muted-foreground`
9. **Logo replacement:** no confirmation modal — direct overwrite on file select
10. **Slug field:** always read-only, visually muted, no edit affordance at all
