# Phase 4.5: Conversion & Qualification - Context

**Gathered:** 2026-06-09
**Status:** Ready for planning

<domain>
## Phase Boundary

Cette phase livre les mécanismes de conversion Basic → Premium :
1. **Verrou 3 leads gratuits** — les 3 premiers leads d'un pro Basic sont accessibles avec coordonnées complètes. Au 4ème, le paywall s'affiche (même comportement que Phase 4, déclenché plus tôt).
2. **Free trial 14j** — la page `/espace/premium` propose un essai gratuit 14 jours via Stripe (`trial_period_days: 14`). Mensuel uniquement pour l'instant (pas de toggle annuel).
3. **Score de qualification automatique** — calculé à la soumission d'un projet, stocké en DB, affiché comme badges ✓/✗ par critère sur chaque lead card.
4. **Bloc "Marché local"** — widget sur `/espace/leads` affichant le nombre de projets dans la zone ce mois et les catégories dominantes.
5. **Profil public éditable** — nouvelle page `/espace/profil` permettant au pro de modifier bio, spécialités, zone et logo d'entreprise. Changements immédiats sur le profil public `/pro/{dept}/{slug}-{id}`.
6. **ROI copy Premium** — refonte de la page `/espace/premium` : hero "Premier contact exclusif" + section ROI chiffrée.
7. **Analytics admin** — mesure des conversions quand un pro qualifié atteint le paywall.
8. **CRM Minimaliste** — Actions directes (Appeler/Email) sur les cartes des leads débloqués, affichage d'un aperçu de la description, et sélecteur de statut (`new`, `contacted`, `won`, `lost`) synchronisé avec l'API.

</domain>

<decisions>
## Implementation Decisions

### Verrou 3 leads gratuits

- **D-01:** Le compteur est **cumulatif** — les 3 premiers leads d'un pro Basic sont gratuits pour toujours. Le compteur ne se réinitialise pas chaque mois.
- **D-02:** Le compteur est stocké dans un champ `free_leads_used` (integer, défaut 0) sur la table `professionals`. Il est incrémenté à chaque premier accès à un lead en mode Basic (lead non encore accédé gratuitement par ce pro).
- **D-03:** La logique `maskLead` devient : `isPremium || (lead fait partie des 3 premiers pour ce pro) || (unlocked_at IS NOT NULL && unlocked_at <= NOW())`. Le planner doit déterminer si on identifie les "3 premiers leads" par rang chronologique ou par flag DB.
- **D-04:** Quand le pro est Premium, le compteur `free_leads_used` est **ignoré** — tous les leads sont débloqués. Si le pro revient en Basic (annulation), le compteur reprend depuis sa valeur actuelle (pas de reset).
- **D-05:** Au 4ème lead et au-delà (pour un Basic sans leads débloqués via 72h) : card floutée + bouton "Passer Premium" — même comportement que Phase 4, aucun overlay ou modal supplémentaire.

### Free Trial Stripe

- **D-06:** **Pas de toggle annuel pour Phase 4.5** — mensuel uniquement. Simplification : 1 seul Stripe Price ID. Le plan annuel est déferré.
- **D-07:** Free trial : **CB requise au démarrage** (Stripe standard `trial_period_days: 14`). Pas de checkout sans CB. La CB est enregistrée, la facturation démarre automatiquement après 14 jours si pas d'annulation.
- **D-08:** Avertissement J-2 avant facturation : **email Stripe automatique** (configurable dans le Stripe Dashboard, aucun développement côté application requis).
- **D-09:** Échec de paiement en fin de trial : **retour silencieux en Basic** — `invoice.payment_failed` → `subscription_status = 'unpaid'` → traité comme Basic. Comportement déjà implémenté Phase 4, pas de nouvelle logique UI nécessaire.

### Score de Qualification

- **D-10:** Le score est **stocké en DB** — champ `qualify_score` (integer 0-4) sur la table `projects`. Calculé une fois à la soumission via `POST /api/v1/projects`.
- **D-11:** Les **4 critères de qualification** :
  1. `budget_range` non null
  2. `customer_phone` non null
  3. `description` de plus de 50 caractères
  4. `customer_email` déjà présent dans un autre projet (client récurrent sur la plateforme)
- **D-12:** Affichage sur la lead card : **icônes ✓/✗ par critère** (4 icônes compactes). SVG inline, couleur `text-foreground` pour ✓ et `text-muted-foreground` pour ✗. Pas de badge texte synthétique.
- **D-13:** Les leads peu qualifiés sont **affichés pareil** que les autres — pas de tri par score, pas de différenciation visuelle globale. Le score est informatif, pas discriminant.

### Profil Public Éditable

- **D-14:** Nouvelle page `/espace/profil` avec les **champs éditables suivants** :
  - Bio / présentation (texte libre)
  - Spécialités / catégorie principale
  - Zone d'intervention (villes ou codes postaux couverts)
  - Photo de logo d'entreprise (recommandée comme logo, pas photo personnelle)
- **D-15:** Le **slug est figé** — généré à la création du profil, jamais modifiable depuis `/espace/profil`. L'URL `/pro/{dept}/{slug}-{id}` reste stable (pas besoin de redirect 301 pour Phase 4.5).
- **D-16:** Les modifications sont **immédiates** — publiées sans validation admin. Modération post-publication si abus constaté.
- **D-17:** Photo de logo : **5 MB max, tous formats image acceptés** (JPG, PNG, WebP, GIF, SVG). Stockage R2 via presigned URL (même pattern que les docs Kbis/décennale Phase 3).

### Claude's Discretion

- Layout exact du bloc "Marché local" sur `/espace/leads` (design et requête SQL)
- Requête SQL pour le comptage de projets par zone/mois et catégories dominantes
- Copywriting exact de la page `/espace/premium` hero + section ROI (respecter les messages clés : "Premier contact exclusif", "1 chantier signé rembourse 6 mois d'abonnement")
- Métriques exactes pour les analytics admin de conversion paywall
- Ordre d'affichage des leads dans le dashboard (plus récent en premier par défaut, comme Phase 4)
- Validation Zod pour les champs profil éditable (longueur max bio, format zone)

</decisions>

<specifics>
## Specific Ideas

- Le logo est recommandé comme "logo d'entreprise" dans le label UI (pas "photo de profil") — cohérent avec le positionnement B2B de la plateforme.
- Le message ROI clé : **"1 chantier signé rembourse 6 mois d'abonnement"** — à intégrer littéralement dans la section ROI de `/espace/premium`.
- Le hero premium : **"Premier contact exclusif"** — accroche principale axée exclusivité, pas "accès aux coordonnées".
- Le 4ème critère de qualification "client récurrent" (email déjà en DB) est un signal fort d'intention — les pros doivent comprendre ce que signifient les 4 badges sur la card.

</specifics>

<deferred>
## Deferred Ideas

- **Champ "nature du projet"** (maison principale vs investissement locatif) dans le simulateur — signal de qualification utile mais nécessite une modification du formulaire Phase 2. Déferré Phase 5+.
- **Demande d'info supplémentaire** sur leads peu qualifiés (pro peut demander à l'admin de compléter le dossier) — déferré Phase 5.
- **Toggle annuel / plan annuel** — déferré jusqu'à validation du taux de conversion mensuel.
- **Multi-catégories par pro** sur le profil — déferré Phase 5 (un pro = une catégorie en Phase 4.5).
- **Portail Stripe** (annulation self-service, mise à jour CB) — déferré si le volume pro le justifie.
- **Rayon km / filtre géographique** sur les leads — déferré Phase 5+ (scale multi-villes).
