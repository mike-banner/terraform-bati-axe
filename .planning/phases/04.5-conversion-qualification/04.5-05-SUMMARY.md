## What was built

### server/api/v1/pro/profile/me.get.ts (NEW)
Authenticated GET — returns `id, canonical_slug, short_id, postal_code, category, bio, zone, logo_url, company_name, full_name, is_verified, subscription_status` + computed `dept` (postal_code.slice(0,2)).

### server/api/v1/pro/profile/me.patch.ts (NEW)
Zod `.strict()` schema — canonical_slug, subscription_status, etc. are hard-rejected (D-15, T-04.5-15/19). Editable fields: `bio` (max 500), `zone` (max 200), `category` (enum), `logo_url` (URL). Returns refetched profile after update.

### server/api/v1/pro/profile/logo-presign.post.ts (NEW)
Mirrors `documents/presign.post.ts` pattern using `aws4fetch` AwsClient. Zod enum: `['image/jpeg', 'image/png', 'image/webp', 'image/gif', 'image/svg+xml']`. 5 MB cap via `Content-Length` header in signed request. Returns `{ signedUrl, fileKey, publicUrl }`. Note: SVG is in allow-list per D-17 — serving with `Content-Disposition: attachment` or CSP proxy is a known follow-up (T-04.5-17).

### server/api/v1/pro/profile/[slug].get.ts (EXTENDED)
Added `bio, zone, logo_url` to SELECT and returned pro object (D-16: immediate public reflection).
