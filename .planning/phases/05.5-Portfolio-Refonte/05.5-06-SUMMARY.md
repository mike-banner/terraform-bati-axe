## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Vitest/oxc ne compilait pas les composants `.vue` de `app/`**
- **Found during:** Task 1 (test de rendu de `RealisationCard.vue`)
- **Issue:** `npx vitest run tests/ui/realisation-card.test.ts` échouait avec `[TSCONFIG_ERROR] Failed to load tsconfig for 'app/components/RealisationCard.vue': Tsconfig not found`. Le `tsconfig.json` racine ne contient que des `references` vers `.nuxt/tsconfig.*.json` (générés uniquement par `nuxt prepare`, absents dans ce worktree exécuteur), et `tsconfig.test.json` (utilisé par `vitest.config.ts` pour éviter justement de dépendre de `.nuxt/`) n'incluait que `tests/**`, `server/utils/**`, `server/api/**` — pas `app/`.
- **Fix:** Ajout de `app/tsconfig.json` (compilerOptions autonomes, sans dépendance à `.nuxt/`) + extension de l'`include` de `tsconfig.test.json` pour couvrir `app/components/**/*.vue` et `*.ts`.
- **Files modified:** `app/tsconfig.json` (nouveau), `tsconfig.test.json`
- **Verification:** `npx vitest run tests/ui/realisation-card.test.ts` passe (3/3) ; `npm run test` global : 47 passent, 2 échecs pré-existants sans rapport (badges.test.ts, hors scope de ce plan).
- **Committed in:** `82941fa` (Task 1 commit)

**2. [Rule 2 - Missing Critical] Endpoint admin manquant pour activer le showcase (pas seulement le désactiver)**
- **Found during:** Task 2 (toggle showcase console admin)
- **Issue:** Le plan suggérait de charger les réalisations via l'endpoint public `/api/v1/completed-projects`, mais celui-ci ne renvoie que les lignes `is_showcased = true`. Sans liste des réalisations non-showcased, l'admin ne pourrait techniquement que retirer une mise en avant existante, jamais en ajouter une nouvelle — contredisant la vérité attendue du plan ("L'admin peut basculer... sur une réalisation", bidirectionnel).
- **Fix:** Ajout de `server/api/v1/admin/realisations.get.ts`, calqué sur le pattern exact de `admin/projects.get.ts` (auth + garde `role === 'admin'` + `serverSupabaseServiceRole`), renvoyant toutes les réalisations avec leur statut `is_showcased` et le nom du pro propriétaire.
- **Files modified:** `server/api/v1/admin/realisations.get.ts` (nouveau), `app/pages/admin/index.vue`
- **Verification:** `grep -q "realisations-showcase"` et `grep -q "Mettre en avant sur la page d'accueil"` passent ; `npm run test` reste vert (hors 2 échecs pré-existants).
- **Committed in:** `a0ab8c2` (Task 2 commit)
