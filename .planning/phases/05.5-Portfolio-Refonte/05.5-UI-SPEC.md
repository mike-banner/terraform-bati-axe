# Phase 05.5 — UI Design Contract

> Visual and interaction contract for Portfolio Pro, Refonte Profil & Social. Extends the Phase 04.7 design system verbatim (`.planning/phases/04.7-UI-Refonte/04.7-UI-SPEC.md`) — no new visual language introduced.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-vue (`components.json` present, style `reka-nova`) — used only for the base `ui/button` primitive; all new UI in this phase is hand-built Vue with existing `.card` / `.bento-card` / `.modal` CSS component classes, not new shadcn blocks |
| Preset | `reka-nova`, baseColor `neutral`, cssVariables true — superseded visually by MASTER.md tokens in `app/assets/css/tailwind.css` (DM Sans + Industrial Grey/Safety Orange, not Geist/neutral) |

This phase introduces the first lucide imports: `Heart` (like button), `ArrowLeft` (immersive back button), `Upload`, `ImagePlus`, `X` (form/gallery controls). Fixed size `w-5 h-5` (16px/20px contexts) or `w-6 h-6` (breadcrumb/back button), never mixed within one component. |
| Font | `DM Sans` (`--font-sans` / `--font-heading`), loaded via Google Fonts `@import` in `tailwind.css` — reuse as-is, no new font |

## Spacing Scale

Reuse the existing token set verbatim from `app/assets/css/tailwind.css` (`--space-*`). No new tokens introduced.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, like-counter inline padding |
| sm | 8px | Compact spacing (gallery thumbnail gaps, form field gaps) |
| md | 16px | Default element spacing |
| lg | 24px | Card padding (gallery cards, add-realisation modal sections) |
| xl | 32px | Section padding (immersive profile sections) |
| 2xl | 48px | Major section breaks (between profile hero and Bento content) |
| 3xl | 64px | Page-level spacing (landing "Chantiers Réalisés" section top/bottom) |

Exceptions:
- Floating back button on immersive profile: `top-4 left-4` (16px) fixed offset from viewport edge — matches `ui-ux-pro-max` "floating navbar" rule (never flush to `top-0`).
- Like-button touch target: 44×44px minimum hit area (accessibility), even though the visual heart icon is 20px — padding absorbs the difference.

## Typography

Reuse Phase 04.7 system verbatim. No new sizes/weights for this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / meta (city, date, like count) | 14px | 500 (medium) | 1.4 |
| Heading (realisation title, section title) | 28px (mobile) → clamp to 40px (desktop) | 700 (bold) | 1.2 |
| Display (immersive profile hero name / company) | 40px (mobile) → clamp to 64px (desktop) | 900 (black), `tracking-tighter` | 1.1 |

Weights used project-wide: 400 and 700/900 for emphasis — do not introduce 500/600 as a third weight tier beyond the existing label use already present in 04.7 components (PremiumBadge, LeadAge already use 500/600 for badges; this phase does not add a fourth weight).

## Layout & Component Contract (phase-specific, extends 04.7 patterns)

### 1. Immersive Public Profile (`/pro/[dept]/[slug]`)
- `definePageMeta({ layout: false })` — or a new minimal `app/layouts/immersive.vue` containing zero `<header>`/nav markup (ui-test in RESEARCH.md explicitly checks for absence of default-layout `<header>`).
- Floating back button: fixed `top-4 left-4`, circular (`rounded-full`), white bg + `shadow-lg`, `ArrowLeft` lucide icon, `hover:-translate-y-0.5` (same easing as `.bento-card`). Navigates to `/`.
- Optional dynamic breadcrumb variant (reuse existing `IdentityBreadcrumbs.vue` pattern) may replace the floating button on desktop ≥768px if the executor prefers; mobile always uses the floating button (never a full breadcrumb bar on mobile — no room, no navbar).
- Content structure: hero block (logo, company name as Display text, badges — reuse `BadgeEntrepriseVerifiee` / `BadgeDecennaleCertifiee` unchanged) → Bento grid (bio, SIRET info, categories) → "Réalisations" gallery section.
- Gallery: reuse `.bento-card` class for each realisation card (`rounded-3xl`, hover elevation). Card shows first image, title, city, like count + heart button (bottom-right overlay on the image, `bg-white/90` chip per ui-ux-pro-max "glass card light mode" rule — never `bg-white/10`).
- Mobile-first: single column gallery `grid-cols-1` below 640px, `grid-cols-2` at `md`, `grid-cols-3` at `lg` — no horizontal scroll at any breakpoint (375/768/1024/1440 verified per 04.7 precedent).

### 2. Backoffice — Add Realisation (`/espace/profil`)
- Entry point: `.btn-primary` button "Ajouter une réalisation" placed in the existing profil page, above or beside the current bio/category form.
- Form surface: **modal** (reuse existing `.modal` + `.modal-overlay` CSS classes already defined in `tailwind.css`) — not a dedicated route. Ponytail: two CSS classes already exist for this, a new page adds routing + layout overhead for no benefit.
- Fields: title (text input, `.input` class), description (textarea, `.input` class), city (text input, prefilled from profile `zone` if available), multi-file image picker (drag/drop or native `<input type="file" multiple accept="image/*">`), each file shows individual upload progress (reuse `UploadProgress.vue` pattern, looped per file — same fix as the logo presign bug: no forced `Content-Length` header in `aws.sign()`).
- Submit CTA: "Publier la réalisation". Cancel: plain text "Annuler".
- Multi-upload UX: files upload in parallel (not sequential), each shows its own thumbnail + progress bar + failure retry icon (`X` to remove before submit).

### 3. Admin — Showcase Toggle
- Reuse existing admin table/list row pattern (consistent with 04.7-07 admin console styling — orange for primary actions). Toggle switch or checkbox, label "Mettre en avant sur la page d'accueil", instantly persists on change (optimistic, matches existing admin verify-doc pattern).

### 4. Landing Page — "Chantiers Réalisés" section (`app/pages/index.vue`)
- Inserted as a new Bento-style section between existing sections, following the same `.reveal` / `.reveal-item` stagger animation already used elsewhere on the page.
- Implementation: **CSS-only scroll-snap carousel** (`overflow-x-auto scroll-snap-x-mandatory` + `scroll-snap-align: start` on cards) — no new JS dependency. Ponytail: Embla is not installed; native CSS scroll-snap is SSR-safe by construction (pure markup, zero hydration risk, zero CLS) and is the smaller diff. Add `embla-carousel-vue` only if the executor later needs programmatic autoplay/dots beyond native scroll — not required by the SPEC.
- Cards: same `.bento-card` gallery card component as the public profile gallery (shared component, not duplicated markup).
- Like button: optimistic UI — heart fills orange immediately on click, count increments client-side, request fires in background; on failure, revert silently (see copywriting contract above).
- Anti-spam for likes: server-side IP hash (SHA-256 of remote IP + project id, no raw IP stored) unique-constraint on `likes` table to block duplicate inserts, combined with a `localStorage` flag client-side to grey out/disable the heart after a successful like on that device (prevents obvious re-click spam without needing auth). Ponytail: no Edge Function, no CAPTCHA — a unique DB constraint is the smallest correct mechanism for this stakes level.
- SEO: section is server-rendered (no client-only carousel), `<img>` tags with real `alt` text (`"{title} — {city}"`), no lazy-load above the fold.
