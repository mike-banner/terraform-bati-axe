<decisions>
## Implementation Decisions

### 1. Limites de taille des champs
- **Noms & Sociétés** (`full_name`, `company_name`) : Max 100 caractères.
- **Email** : Max 254 caractères (Standard RFC).
- **Bio** : Max 500 caractères.
- **Messages / Descriptions** : Max 1000 caractères.

### 2. Expérience Utilisateur (UI) & Formats
- Formats attendus affichés via des placeholders explicites (ex: `06 12 34 56 78` pour téléphones, format requis pour décennales).
- Masques de saisie / formatage auto côté front pour empêcher les mauvaises saisies.
- Nettoyage des espaces/caractères invalides avant l'envoi API.
- Compteur de caractères **uniquement** pour les champs longs (bio, description, message). Les autres champs utiliseront un blocage natif (`maxlength`).

### 3. Profondeur de la Validation (Hardening)
- Validation API : Sécurisation absolue via `.max()` et `.regex()` dans tous les schémas Zod.
- Validation DB : Ajout de contraintes `CHECK` via migration SQL (Supabase) pour garantir l'intégrité des données au plus bas niveau.
</decisions>
