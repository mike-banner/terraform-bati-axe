# Audit — Validation des inputs (2026-07-19)

Grep exhaustif sur `app/pages/` + `app/components/` de tous les `<input>` / `<textarea>`
(hors `type="file"`, `checkbox`, `radio`, `hidden` qui n'ont pas de contrainte de format
pertinente). 29 champs texte trouvés au total, répartis sur 9 fichiers.

Méthode : pour chaque champ sans contrainte client, le schéma Zod serveur correspondant a
été vérifié pour savoir si la borne haute existe déjà côté serveur (auquel cas le fix
client doit juste la refléter) ou si elle manque aussi côté serveur (fix à faire des deux
côtés).

---

## Déjà correct (ne pas retoucher sauf régression)

- `app/pages/simulateur.vue` — très bien fait : `description` (minlength=20, maxlength=1000,
  cohérent avec `projects.post.ts` `.min(20)`), `postal_code` (maxlength=5, minlength=5,
  pattern `[0-9]{5}`), `customer_phone` (maxlength=17). Seuls `customer_name` et
  `customer_email` n'ont pas de maxlength (voir plus bas).
- `app/pages/pro/claim.vue` — `siret` (maxlength=19 + inputmode numeric + normalisation),
  `phone` (maxlength=20 + normalisation), `postal_code` (maxlength=5 + inputmode numeric).
- `app/pages/espace/profil.vue` — `bio` (maxlength=500), `zone` (maxlength=200).
- `app/pages/admin/index.vue` — les deux inputs sont `type="date"` et `type="checkbox"`,
  rien à ajouter.
- `app/pages/espace/dashboard.vue` — `expirationDate` est `type="date"`, rien à ajouter.
