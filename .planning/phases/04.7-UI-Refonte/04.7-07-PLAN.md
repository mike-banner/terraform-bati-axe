**Accessibility Audit (WCAG AA):**

For each page (index, simulateur, pro/claim, pro/profile, espace/profil, espace/leads/index, espace/leads/[id], espace/premium, pro/{dept}/{slug}, admin/index, legal/*):

1. **Contrast Verification (WCAG AA = 4.5:1 text, 3:1 non-text):**
   - Use DevTools > Accessibility > Color Contrast (or WebAIM Contrast Checker)
   - Check all text colors against background colors
   - Common checks:
     - Body text (Noto Sans 400, 14-16px, #374151) on #ECFEFF → passes
     - Links (cyan #0891B2) on white → passes
     - Labels (Figtree 500, 14px, #164E63) on #ECFEFF → passes
     - Button text (white) on green (#22C55E) → passes
     - Button text (cyan #0891B2) on white → passes
     - Disabled text (#94A3B8) on #F1F5F9 → check (may be 3:1)
   - Document each contrast ratio found; flag any < 4.5:1 for text

2. **Focus States (must be visible for keyboard nav):**
   - Tab through each page: all buttons, inputs, links must have visible focus ring (outline or ring)
   - Check for outline: 2px solid #0891B2, outline-offset 2px (or ring-primary from Tailwind)
   - Verify focus is not outline: none (hidden)
   - Test with keyboard: Tab, Shift+Tab, Enter, Space

3. **Heading Hierarchy (h1 > h2 > h3, no gaps):**
   - Each page should have exactly one h1 (main title)
   - h2 follows h1 (no h3 without h2)
   - Semantic: headings use h1-h6 tags, not divs with large text
   - Check: landing (h1: "Trouvez l'artisan..."), claim (h1: form title), etc.

4. **Form Accessibility (labels, error messages, required markers):**
   - All input fields must have associated labels (label for="id" or wrapped)
   - Error messages must be linked to inputs (aria-describedby)
   - Required fields must be marked (asterisk or aria-required="true")
   - Fieldsets group related inputs (radio groups, checkboxes)

5. **Link Accessibility (descriptive text, not "click here"):**
   - All links must have descriptive text ("Read our privacy policy" not "click here")
   - Links must have focus ring visible
   - Links should have underline on hover (visual feedback)

6. **Image Alt Text:**
   - All images must have alt text (alt="description")
   - Decorative images can have alt="" (empty)

7. **Lighthouse Accessibility Audit:**
   - Run Lighthouse on each page: DevTools > Lighthouse > Accessibility
   - Target: score 90+
   - Document any issues found (e.g., missing alt, low contrast, missing labels)

**Responsive Breakpoint Testing (375px, 768px, 1024px, 1440px):**

For each page, test layout at four breakpoints:

1. **375px (Small Mobile):**
   - Single-column layouts (grids should stack)
   - Touch-friendly button/input sizes (48px minimum height)
   - No horizontal scroll
   - Text is readable (not cramped)
   - Navigation is accessible (menu visible or hamburger)

2. **768px (Tablet):**
   - 2-column layouts should appear (if applicable)
   - Spacing scales up (padding, margins increase)
   - Cards should show 2 per row (if grid)
   - No horizontal scroll
   - Desktop navigation should be visible

3. **1024px (Desktop):**
   - Full layout with 3+ columns (if applicable)
   - Maximum spacing (padding, gaps at full size)
   - Full hero/hero images visible
   - Sidebar layouts visible
   - No horizontal scroll

4. **1440px (Large Desktop):**
   - Layout should not expand further (max-width container)
   - All elements within viewport
   - No awkward gaps or stretching
   - No horizontal scroll

Test each page systematically using DevTools Device Toolbar:
- Landing page: hero splits 1 → 2 columns? Categories visible? CTAs accessible?
- Pro claim: form stacks mobile, side-by-side tablet?
- Leads grid: 1 → 2 → 3 card columns?
- Detail page: card sections scale properly?
- Premium page: pricing cards responsive?
- Admin table: scrolls on mobile? Header visible?
- Legal pages: single column at all breakpoints? Padding scales?

Document any layout issues (cards overlapping, text overflow, missing elements).

**Pre-Delivery Checklist (14 items from UI-SPEC.md lines 366–387):**

Create document checking each item:

- [ ] No emojis used as icons (use SVG from Heroicons/Lucide only)
- [ ] All icons from consistent icon set
- [ ] `cursor: pointer` on all clickable elements
- [ ] Hover states with smooth transitions (150-300ms)
- [ ] Light mode: text contrast 4.5:1 minimum (WCAG AA)
- [ ] Focus states visible for keyboard navigation (outline or ring)
- [ ] `prefers-reduced-motion` respected (no animation when disabled)
- [ ] Responsive tested at: 375px, 768px, 1024px, 1440px
- [ ] No content hidden behind fixed navbars (top margin applied)
- [ ] No horizontal scroll on mobile
- [ ] Color palette matches MASTER.md exactly
- [ ] Typography uses Figtree + Noto Sans only
- [ ] Spacing follows spacing system (`--space-*` tokens)
- [ ] Buttons have visible focus states
- [ ] Forms fully keyboard accessible
- [ ] Images have alt text
- [ ] No console errors or warnings

For each item, test and document:
- Pass / Fail
- Where tested (which pages)
- Remediation notes (if fail)
