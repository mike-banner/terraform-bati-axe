## Finding critique n°1 — le token cascade

`app/pages/index.vue`, `pro/claim.vue`, `espace/profil.vue`, `simulateur.vue`, `mon-projet/[token].vue`,
`admin/index.vue`, `espace/leads/[id].vue`, `espace/leads/index.vue` n'utilisent **presque jamais** de
classes Tailwind brutes de couleur (`cyan-*`, `green-*`). Ils consomment les classes sémantiques
`bg-cta`, `text-primary`, `bg-background`, `text-text`, `border-border`, `bg-page` définies dans
`app/assets/css/tailwind.css` (`@theme inline` + `:root`). **Redéfinir ces 6 tokens dans le fichier CSS
suffit à re-thémer la couleur de ~10 pages sans toucher un seul fichier `.vue`.**

Ce que le token swap NE couvre PAS (nécessite édition fichier par fichier) :
- Le radius (`rounded-md`/`rounded-lg` actuels → `rounded-3xl` bento demandé par l'UI-SPEC) — aucune variable `--radius-*` n'est utilisée dans les classes Tailwind des pages (elles sont hardcodées `rounded-md`/`rounded-lg` littéralement, pas `rounded-[var(--radius-lg)]`). **Chaque page doit être éditée pour le radius.**
- La typographie (échelle massive `text-6xl`→`text-8xl`, `tracking-tighter`, weight 900) — `--font-heading: 'Figtree'` doit devenir Satoshi/DM Sans, et les tailles de titres (`text-4xl`, `text-5xl` actuels) doivent grossir.
- Le layout Bento Grid du hero (actuellement un split texte/photo 2 colonnes classique, pas un bento 12 colonnes).
- Les micro-interactions demandées (`hover:-translate-y-1`, `shadow-safety/20`, stagger reveal) — le hero d'`index.vue` a déjà un `.reveal`/`.reveal-item` stagger scoped-style, réutilisable comme squelette d'animation mais à généraliser.

## Pattern Assignments

### `app/assets/css/tailwind.css` (config, transform)

**Rien à copier ailleurs — c'est la source.** Tokens actuels à remplacer (lignes 12-21, dupliqués lignes 90-100) :

```css
--color-primary: #0891B2;    /* cyan → devient industrial #64748B */
--color-secondary: #22D3EE;  /* cyan clair → à redéfinir (ex: slate-400 ou supprimé) */
--color-cta: #22C55E;        /* vert → devient safety #F97316 */
--color-background: #FFFFFF; /* inchangé */
--color-page: #ECFEFF;       /* cyan clair fond → devient slate-50 #F8FAFC */
--color-text: #164E63;       /* dark cyan → devient slate-900 #0F172A / slate-700 */
```

Radius actuels (lignes 36-39) à étendre pour le bento : ajouter `--radius-2xl`/`--radius-3xl` ou utiliser
directement les classes Tailwind `rounded-3xl` dans les pages (le fichier n'a pas de variable au-delà de
`--radius-xl: 16px`).

Font-family (ligne 4, import Google Fonts Figtree/Noto Sans + lignes 9-10) à remplacer par Satoshi/DM
Sans — vérifier `app/assets/` pour fonts auto-hébergées existantes avant d'ajouter un nouvel import
(voir CONTEXT.md discretion : self-hosting déjà en place ailleurs dans le projet).

Le bloc `@layer components` (lignes 217-313 : `.btn-primary`, `.btn-secondary`, `.card`, `.input`,
`.modal`) consomme déjà les variables `--color-cta`, `--color-primary`, `--radius-*` — donc ces classes
utilitaires se re-thèment automatiquement une fois les tokens changés, mais leurs `border-radius`
restent `--radius-md`/`--radius-lg`/`--radius-xl`, pas `3xl` — à vérifier si `.card` doit devenir bento
(sinon laisser `.card` pour usages secondaires et utiliser des classes Tailwind directes `rounded-3xl`
pour les nouveaux bento blocks).
